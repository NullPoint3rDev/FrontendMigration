BetaFrontWT

Фронтенд-приложение WeldTelecom — система учёта и мониторинга сварочного оборудования, сварщиков и отчётности.

Стек технологий

- React 18 — UI
- Vite 4 — сборка и dev-сервер
- React Router 6 — маршрутизация
- Material-UI (MUI) 5 — компоненты и темы
- Chart.js / react-chartjs-2 — графики
- react-icons — иконки
- xlsx — работа с Excel

Требования

- Node.js 16+ (рекомендуется 18+)
- npm или yarn

Установка

bash
Клонирование репозитория (если ещё не склонирован)
git clone <url-репозитория>
cd BetaFrontWT

Установка зависимостей
npm install


Запуск

bash
Режим разработки (локально, порт 3002)
npm run dev
или
npm start

Запуск с доступом по сети (0.0.0.0)
npm run start-external

Сборка для продакшена
npm run build

Просмотр собранной сборки
npm run preview


По умолчанию приложение доступно по адресу: http://89.109.8.59:3002

Настройка API

Бэкенд подключается через переменную окружения или настройки Vite:

- Базовый URL API: по умолчанию `http://89.109.8.59:8085/api`
- В режиме разработки запросы к /api проксируются на бэкенд (см. `vite.config.js` и `package.json` → `proxy`)

Чтобы использовать другой хост/порт бэкенда, измените в `vite.config.js`:

- `server.proxy['/api'].target`
- `define['process.env.REACT_APP_API_URL']`

И при необходимости в `src/services/api.js` — `API_BASE_URL`.

Основные разделы приложения

- Главная — дашборд
- Предприятие — подразделения, сотрудники, сварщики, карта предприятия
- Ресурсы — сварочное оборудование, сетевое оборудование, материалы, WPS
- Мониторинг — карта предприятия, интерактивная карта
- Отчёты — шаблоны отчётов, формирование отчётов (по сварщикам, по расходу проволоки и др.)
- Уведомления — настройка уведомлений
- Настройки — параметры системы
- Профиль — профиль пользователя

Доступ к разделам защищён: при отсутствии токена выполняется перенаправление на страницу входа (`/login`).

Авторизация

- Страница входа: `/login`
- При ответе 401 от API токен сбрасывается и выполняется редирект на `/login`
